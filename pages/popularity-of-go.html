Popularity of Go

The popularity of Go can be attributed to many factors, but many stem from the history of its development.

When Go was announced in 2009, some people were excited for a new Google-backed language, but many were skeptical \[JOSHI]. The few that were excited was because of decisions that the creators of Go made while designing this language.

In 2007, the three eventual primary creators of Go—Robert Griesemer, Rob Pike, and Ken Thomson—expressed their frustrations with each other concerning the technologies the three were using at Google \[JOSHI], \[PIKE]. The current technologies they were using struggled with slow build times, obfuscated and lengthy algorithms, and difficulty writing automated tools \[PIKE].

Most of the programming languages that major tech corporations use were designed for an era that had already passed. Things like multicore processors, giant networked systems, huge computation clusters, and modern web programming models were recent additions that older languages do not handle conveniently.

Robert Griesemer, Rob Pike, and Ken Thomson needed a modern language built to handle the requirements of the present.

They designed Go to have a garbage collector, built-in concurrency, effective dependency management, quick development time, and easy to use interfaces between components \[PIKE].

Go's garbage collector allows developers to write code without having to manage memory manually. This increases developer velocity allowing them to write business logic instead of preventing memory leaks.

Go has built-in concurrency models such as goroutines for running concurrent functions, channels for sending data between different goroutines, and other data models for ensuring proper code execution. All of these tools work together to provide developers one of the best experiences working with concurrent code.

All programming languages need a way to import external libraries to use with their code. Go makes use of the internet to import the packages you need. This makes it simple for anyone anytime to get the libraries they need.

To further extend developer velocity while using Go, they made sure to keep annoyances few and far between. The designers did this in a multitude of ways, but the net effect of which is frustration-free development.

Another thing Go does well is how simple it is to interact for foreign components. Go makes this simple by making having an easy-to-use interface system that allows functions to accept inputs that fit to a specific pattern. Many languages have a similar system, but Go makes it obvious what's happening and doesn't try to hide things from the developer.

All of these design decisions were necessary and contributed toward Go's growth and success.

Version 1.0 of Go released in 2012. This is when the "Go 1" announcement was made that promised stability as future versions are released. All working code made with version 1.0 will work with 1.1, 1.2, 1.3, etc. \[Go 1]. This guarantee of stability was another factor in the growing popularity of this language.

The first major open source application made with Go was Docker released in 2013 \[dotcloudtv]. Docker is currently the most popular Cloud development technology \[2025 Dev Surv]. Docker is an application that isolates and runs other applications through containers. By doing this, an application relies less on the machine it is ran on and allows for a more streamlined method for hosting applications \["What is Docker"].

Another boost that Go had was another major open source application Kubernetes (K8s) 1.0 released in 2015 \[LARDINIOS]. K8s is an application that works with Docker to manage containers on a mass scale.

With two major open source applications being developed with Go, there were now major projects backing this language. These applications pushed Go further into the spotlight of the programming language of the cloud.

Despite all of the success Go has had since release, it was still struggling because it did not support generic functions—functions that do not rely on a specific type but rather a category of type that all yield to a specific standard. This was an annoyance among Go developers until release go1.18 in 2022 \[The Go Team]. There was a surge of develops using go around 2022, likely because of this change \[Fig. X] \[Table X].
