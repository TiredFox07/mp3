Popularity of Go

The popularity of Go can be attributed to many factors.

When Go was announced in 2009, some people were excited for a new Google-backed language, but many were skeptical \[JOSHI]. Those that were excited were because of decisions that were made during the conception of this language.

In 2007, Robert Griesemer, Rob Pike, and Ken Thompson expressed their frustrations with each other concerning the current technologies they were at Google. \[JOSHI], \[PIKE]. Some specific pain points were slow build times, poor program understandably, and difficulty writing automatic tools \[PIKE]. On top of this, most languages at the time did not handle the of multi-core processors, massively networked systems, huge computation clusters, and modern web programming models \[PIKE]. They wanted a modern language that was built to function during the modern era.

Go was designed with a garbage collector for less developer overhead, built-in and effective concurrency model that works well with multi-core processors, better methods of managing dependencies compared to the current environment, allow for architecture to adapt and change quickly and with as few difficulties as possible, and allowing effective methods to communicate between components \[PIKE]. This design was critical for Go's growth and success.

With version 1.0 being released in 2012, the larger programming community realized that this wasn't just some one-off language that Google was experimenting with—it was a language that was going to remain stable throughout its development \[Go 1].

Evidence of Go's newfound trust was evident when Docker—which is currently the most popular Cloud development technology \[2025 Dev Surv]—was built and released in 2013 using Go \[dotcloudtv], \["What is Docker"]. Docker runs applications in a container, meaning if it works on one machine, it should work on any machine. Docker's growth was accelerated further after the release of Kubernetes 1.0 in 2015 \[LARDINIOS]. Because Docker and Kubernetes (K8s) are both built using Go, these projects further pushed Go into the spotlight of the cloud's programming language.

Though from release, Go has been struggling from a lack of generic functions—functions that could be called without specifying a type. This was a major annoyance among Go developers. Though, for release go1.18 in 2022, that all changed \[The Go Team]. Generics were finally added into the language specification. When looking at 2022, you can see a major jump in Go developers, likely because of how impactful this change was.
