Developer Tooling

When you download Go, you do not only get the compiler; you get a variety of tools that help contribute to making writing Go as efficient as possible.

The first major tool is gofmt. gofmt is a tool that automates the formatting of all Go code. This tool eliminates style debates between developers. The correct format is the format given when you run gofmt. Pike says that he and many others within the Go community believe that this is the best tool for Go despite not actually being part of the language [3].

Another major tool used among Go developers is go test. go test is a testing framework built right into Go. Many other languages require the use of external testing solutions; Go's is built-in [27]. Having it built into the language removes that initial barrier of needing a testing framework to test code. Beyond that, the testing package provides many of the tools a developer would need to properly test their code. This ties in to Go's philosophy of wanting everything a developer would need out of the box.

Other languages like Python and Java require a runtime environment in order to execute programs, however, Go differs because it is packaged with the go build command. go build builds your Go program into a single executable binary that does not rely on external libraries [28]. This single executable removes the dependency on an external program that runs the code for you; it's all bundled into one.

Every sufficiently complex program will eventually depend on code written by someone else or external third-party libraries; Go uses the go mod command to manage project dependencies. Dependencies are stored in a go.mod file that explicitly states what the project depends on [29]. This fixes the problem of having different versions of dependencies, because the versions of the dependencies are centralized into a single file.

Unlike the other tools, a developer never intentionally runs gopls. gopls is the official Go Language Server that allows IDEs to provide services like auto-completion, error highlighting, and jump-to-definition [30]. Because it is developed by the Go team, it is always in sync with the latest version of the language. This reinforces the Go toolchain theme that everything you could need to develop a Go program is available out of the box.

These tools work in conjunction to provide the best experience a developer could have immediately after downloading. This is at the core of Go's philosophy: simplicity. Having everything provided in one simple download, less time is spent on setting up and more time is spent writing the code that will run the project.
